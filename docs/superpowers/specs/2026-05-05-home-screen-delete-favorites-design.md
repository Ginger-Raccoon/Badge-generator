# Удаление проектов и избранное на главном экране

**Дата:** 2026-05-05
**Статус:** Утверждён

---

## Цель

Добавить на главный экран две функции:
1. Удаление проекта с диалогом подтверждения и настройкой "больше не спрашивать"
2. Добавление проектов в избранное с отображением их вверху списка

---

## Хранилище

Новый файл `prefs.json` в папке `BadgeGenerator` (рядом с папками проектов).

Структура:
```json
{
  "favorites": ["Название проекта"],
  "skipDeleteConfirm": false
}
```

- `favorites` — массив строк с именами избранных проектов
- `skipDeleteConfirm` — глобальный флаг, отключает диалог подтверждения для всех проектов

Если файл отсутствует — используются дефолты: `{ favorites: [], skipDeleteConfirm: false }`.

---

## Бэкенд (ipc.js)

Три новых IPC-хендлера:

**`prefs:load`**
Читает `prefs.json`. Если файл не существует — возвращает дефолтный объект. Никогда не бросает ошибку.

**`prefs:save`**
Принимает объект предпочтений, записывает в `prefs.json`.

**`projects:delete`**
Принимает имя проекта, удаляет папку через `fs.rmSync(projectDir, { recursive: true })`.

---

## Preload (preload.js)

Три новых метода в `window.api`:
- `loadPrefs()` — вызывает `prefs:load`
- `savePrefs(prefs)` — вызывает `prefs:save`
- `deleteProject(name)` — вызывает `projects:delete`

---

## Фронтенд (HomeScreen.jsx)

### Состояние

Новые поля в state:
- `favorites: string[]` — список избранных проектов
- `skipDeleteConfirm: boolean` — флаг "не спрашивать"
- `pendingDelete: string | null` — имя проекта ожидающего удаления (управляет диалогом)

При монтировании загружаются `projects` и `prefs` параллельно.

### Список проектов

Проекты делятся на два блока:
1. **Избранные** — отфильтрованные по `favorites`, показываются первыми
2. **Остальные** — все проекты не из избранного

Если есть оба блока — между ними рендерится `<Divider>`.

Каждый элемент списка содержит справа две иконки (всегда видимые):
- **Звёздочка** (`StarIcon` / `StarBorderIcon`) — тоглает избранное, сохраняет `prefs.json`
- **Корзина** (`DeleteIcon`) — инициирует удаление

### Логика удаления

1. Клик по корзине → если `skipDeleteConfirm = true` — удалить немедленно, иначе → открыть диалог (`pendingDelete = name`)
2. В диалоге: заголовок "Удалить «{name}»?", чекбокс "Больше не спрашивать", кнопки "Отмена" / "Удалить"
3. При подтверждении: если чекбокс отмечен — сохранить `skipDeleteConfirm: true` в `prefs.json`, затем вызвать `deleteProject(name)`, обновить список
4. Если проект был в избранном — удалить его и из `favorites`

### Логика избранного

- Клик по звёздочке → добавить/убрать имя из `favorites`, вызвать `savePrefs`
- Список перерисовывается реактивно

### Обработка кликов

Иконки звёздочки и корзины оборачиваются в `<IconButton>` с `e.stopPropagation()`, чтобы клик по иконке не открывал проект.

Структура элемента списка:
```
ListItem
  └─ ListItemButton (onClick → открыть проект)
       ├─ ListItemText (название)
       └─ Box (иконки, ml: auto)
            ├─ IconButton (звёздочка, stopPropagation)
            └─ IconButton (корзина, stopPropagation)
```

## Обработка ошибок

- `prefs:load` — при любой ошибке чтения/парсинга возвращает дефолтный объект, не бросает исключение
- `prefs:save` — ошибки записи логируются в консоль, UI не блокируется
- `projects:delete` — если папка не найдена или удаление не удалось, ошибка пробрасывается в рендерер; HomeScreen показывает уведомление через `Snackbar`

## Критерии готовности

- [ ] Клик по корзине удаляет проект и убирает его из списка
- [ ] При `skipDeleteConfirm = false` показывается диалог подтверждения
- [ ] Чекбокс "Больше не спрашивать" сохраняется глобально — последующие удаления происходят без диалога
- [ ] Клик по звёздочке добавляет/убирает проект из избранного
- [ ] Избранные проекты отображаются выше остальных
- [ ] Состояние избранного и флаг подтверждения сохраняются между перезапусками
- [ ] Удалённый избранный проект исчезает и из избранного
