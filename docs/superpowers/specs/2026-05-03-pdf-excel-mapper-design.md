# Badge Generator — Дизайн-спецификация

**Дата:** 2026-05-03  
**Статус:** Согласовано

## Обзор

Десктопное приложение (Electron + React) для генерации многостраничных PDF из шаблона PDF и файла Excel. Пользователь размечает прямоугольные зоны на шаблоне PDF, привязывает каждую зону к столбцу Excel, затем генерирует итоговый PDF — каждая строка Excel становится отдельной страницей с данными, размещёнными в заданных зонах.

## Основной рабочий процесс

```
Домашний экран (список проектов)
  → Создать / открыть проект
  → Загрузить PDF-шаблон
  → Загрузить Excel файл (читаем заголовки столбцов)
  → Нарисовать зоны на PDF (зажать и тянуть мышью)
  → Привязать каждую зону к столбцу Excel (дропдаун)
  → Нажать "Генерировать" → сохранить output.pdf
```

## Технологический стек

| Задача | Библиотека |
|---|---|
| Десктопная оболочка | Electron + electron-forge |
| UI | React + Vite + Material UI (MUI) |
| Отображение PDF | pdf.js |
| Генерация PDF | pdf-lib |
| Чтение Excel | xlsx (SheetJS) |
| Хранение проектов | JSON файлы на диске |

## Архитектура

### Компоненты

Все UI-компоненты строятся на базе **Material UI (MUI)**: кнопки, дропдауны, списки, диалоги, лейауты — стандартные MUI-компоненты. Кастомная стилизация только там, где MUI не перекрывает потребность (SVG-оверлей зон на PDF).

- **HomeScreen** — список проектов из `~/Documents/BadgeGenerator/`, кнопка «Новый проект»
- **App** — корневой компонент, хранит состояние проекта, переключает между HomeScreen и Editor
- **Editor** — двухколоночный макет: PDFViewer слева, Sidebar справа
- **Toolbar** — кнопка «Генерировать», кнопки загрузки PDF и Excel
- **PDFViewer** — рендерит страницу PDF через pdf.js canvas, SVG-оверлей для рисования и отображения зон
- **ZoneList** — список зон с автогенерированным названием («Зона 1», «Зона 2»...), дропдауном столбца, дропдауном шрифта, кнопкой удаления
- **ZoneRect** — один прямоугольник зоны на SVG-оверлее с подписью

### Состояние (хранится в App)

```ts
interface Project {
  name: string
  templatePdfPath: string
  excelPath: string
  columns: string[]       // читается из заголовков Excel
  zones: Zone[]
}

interface Zone {
  id: string
  label: string
  x: number       // PDF points, начало координат — нижний левый угол
  y: number
  width: number
  height: number
  column: string
  font: 'Roboto' | 'PTSerif'
  fontSize: number  // по умолчанию: 12
}
```

## Хранение проектов

- При первом запуске приложение создаёт папку `~/Documents/BadgeGenerator/`
- Каждый проект — подпапка: `~/Documents/BadgeGenerator/<название-проекта>/project.json`
- Автосохранение при каждом изменении (добавление/редактирование/удаление зоны, загрузка файлов)
- Пользователь никогда не взаимодействует с файловой системой напрямую
- При открытии проекта: проверяем наличие PDF и Excel по сохранённым путям; если файл не найден — предлагаем выбрать его заново

## Система координат

pdf.js и pdf-lib используют разные системы координат. Зоны всегда хранятся в **PDF points** (пространство координат pdf-lib: начало — нижний левый угол, ось Y направлена вверх).

Преобразование при рисовании зоны на канвасе:

```js
const scaleX = pdfPageWidth  / canvasWidth
const scaleY = pdfPageHeight / canvasHeight

const pdfX = zoneX * scaleX
const pdfY = pdfPageHeight - (zoneY + zoneHeight) * scaleY  // инвертируем ось Y
```

Обратное преобразование для отображения — применяем обратные формулы.

## Шрифты

Бандлятся вместе с приложением в `assets/fonts/`:

| Шрифт | Файл | Назначение |
|---|---|---|
| Roboto Regular | `Roboto-Regular.ttf` | без засечек, по умолчанию |
| PT Serif Regular | `PTSerif-Regular.ttf` | с засечками, оптимизирован для кириллицы |

Оба шрифта поддерживают кириллицу в полном объёме. Никакой настройки шрифтов от пользователя не требуется. Шрифт выбирается для каждой зоны отдельно через дропдаун в ZoneList.

## Генерация PDF

```
1. Читаем Excel → массив объектов строк: [{ "Имя": "Иван", "Фамилия": "Петров" }, ...]
2. Загружаем байты шаблона через pdf-lib (PDFDocument.load)
3. Встраиваем оба шрифта в документ
4. Для каждой строки Excel:
   а. Копируем страницу шаблона (pdfDoc.copyPages)
   б. Для каждой зоны: page.drawText(row[zone.column], { x, y, size, font })
5. Собираем все страницы в один PDFDocument
6. Показываем нативный диалог "Сохранить как" → записываем output.pdf
7. Отображаем прогресс в UI: "Генерация 42 / 200..."
```

Если ячейка пустая — текст для этой зоны на этой странице не рисуем.

## Макет UI

```
┌──────────────────────────────────────────────────────────┐
│  [PDF: template.pdf ▼]  [Excel: data.xlsx ▼]  [Генерировать] │
├───────────────────────┬──────────────────────────────────┤
│                       │  Зоны:                            │
│   PDF Viewer          │  ┌────────────────────────────┐  │
│   (canvas + SVG)      │  │ Зона 1 │ Имя    ▼ │Roboto  ▼│  │
│                       │  │ Зона 2 │Фамилия ▼ │PTSerif ▼│  │
│                       │  └────────────────────────────┘  │
│                       │  Нарисуйте зону на PDF           │
└───────────────────────┴──────────────────────────────────┘
```

Рисование зоны: mousedown → drag → mouseup на SVG-оверлее создаёт новую зону и добавляет её в ZoneList.

Клик по зоне в ZoneList подсвечивает соответствующий прямоугольник на PDF.
